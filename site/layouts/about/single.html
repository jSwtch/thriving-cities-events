About this submission

Site performance

This site is super fast! I like to use GTMetrix for performance ratings, you can analyze any URL for comparision. The image shown here is a summary of the results.

Fully responsive design

This site is built with mobile users in mind! We have enabled functional breakpoints for different devices and fluid text for smooth transitions. Furthermore all images have multiple compressed forms for smaller file size on mobile. This translates to smaller page size and a faster load time.

Zero friction

This site is designed for easy implementation. There is no javascript framework or complicated content management system (see why choose static ). A new event is very easy to add. This site uses Go templating to generate html pages, and anyone can add a new event by writing a simple markdown file following the format as shown. This means you may not have to rely on your developers to constantly update the events page. Just write the markdown, and upload to the source. This will trigger a new deployment of the site, and the results will be live within minutes.

Easily themable

This site is built with customization in mind. Font size, color, and most margin/padding can be easily tweaked, or tuned to your liking. CSS custom properties, and variables are used in the main css file so you don't have to search through the code. The image shown here is how some custom color parameters could be adjusted.

Why choose static?

This is all about the right tool for the right job. This type of events page has to be fast, and easily accessible on mobile. The static site gives us the speed and security we need, with no complicated code base, or content management system. Furthermore, new events can easily be added by marketers, or social media coordinators. You don't need to be a developer to keep the site updated. A static solution with template generated markdown, and continious deployment is the perfect fit!

We like everything but...

I completely understand. I think it is important to establish a feedback loop in design and development projects. I am happy to make any changes to the site provided they remain within the scope of the project.

With this solution a new event could be created from a mobile device, that then triggers a redeployment of the whole site.
